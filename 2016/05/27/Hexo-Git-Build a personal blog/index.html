Hexo+Git 搭建个人博客

  • 学习笔记

##配置环境

安装 Node.js （必须）

作用：用来生成静态页面的

到Node.js官网下载相应平台的最新版本，一路安装即可。

安装 Git （必须）

作用：把本地的hexo内容提交到github上去.

没安装Git的推荐看下 廖雪峰的官方网站

申请 Coding

我这里使用的是coding，Coding和Github的使用方式是一样的。

新建项目，注意，博客项目名称必须和用户名一致

配置SSH公匙

在个人账户界面，有个SSH公匙，添加你的SSH公匙，怎么生成SSH公匙我就不再叙述了，网上搜索一下很多。

安装Hexo

Hexo下载地址

新建一个文件夹，右键 选择Git Bash Here

  1. npm install hexo-cli -g
  2. hexo init blog
  3. cd blog
  4. npm install
  5. hexo server

此时本地服务就已经启动成功了，浏览器输入http://localhost:4000

现在Hexo还只是本地服务，我们要配置好hexo，提交到git上面。

现在我们需要配置_config.yml文件，来建立关联

_config.yml 在blog文件夹根目录下，用文本编辑器打开

title: 网站标题

subtitle: 网站副标题

description: 网站描述

author: 姓名

email: 邮箱

language: 语言 zh-CN

timezone:

url: 个人域名

root: /

permalink: :year/:month/:day/:title/

permalink_defaults:

重点

deploy:

type: git

repository: git@git.coding.net:lqs20115/lqs20115.git 这里填写项目的SSH

branch: master

以上步骤都完成之后，就可以部署提交了

部署提交

每次部署提交命令

  1. hexo clean
  2. hexo generate
  3. hexo deploy

执行第三步之后，如果你的SSH公匙设置了密码，就会有个弹出框，输入你创建SSH公匙时创建的密码就OK了，如果你的SSH公匙没有密码，此处会自动跳过

一些常用命令：

hexo new”postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本
